问题3：CACHE 有哪几种地址映射方式？各自的主要特点是什么？

Cache（高速缓存）是计算机系统中重要的组成部分，用于提高CPU访问内存的速度。地址映射是Cache的关键技术之一，它决定了主存中的数据块如何映射到Cache中的行。常见的地址映射方式有以下三种：

  1. 直接映射（Direct Mapping）：

    • 映射方式： 主存块地址除以Cache行数取余，余数即为该主存块在Cache中对应的行号。每个主存块只能映射到Cache中的一个特定行。
    • 特点：
      • 实现简单，硬件开销小。
      • 容易发生冲突，即多个主存块竞争同一个Cache行，导致频繁的替换（抖动）。
      • Cache利用率不高。

  2. 全相联映射（Fully Associative Mapping）：

    • 映射方式： 主存块可以映射到Cache中的任何一行。
    • 特点：
      • 灵活性高，Cache利用率高，不易发生冲突。
      • 实现复杂，需要大量的比较器来并行比较主存块地址和Cache中所有行的标记，硬件开销大。
      • 速度较慢。

  3. 组相联映射（Set Associative Mapping）：

    • 映射方式： 将Cache分为若干个组，每个组包含若干行。主存块地址除以Cache组数取余，余数即为该主存块在Cache中对应的组号。然后，该主存块可以映射到该组中的任何一行。
    • 特点：
      • 介于直接映射和全相联映射之间，是两者的折中方案。
      • 既有一定的灵活性，又不需要过多的比较器，硬件开销适中。
      • 性能较好，应用广泛。例如，2路组相联Cache表示每个组有两行。

问题4：简述集中式总线仲裁有哪三种方式？各有什么特点？

总线仲裁是解决多个设备竞争使用总线的问题的技术。集中式总线仲裁由一个中央仲裁器来管理总线的使用权。常见的集中式总线仲裁方式有以下三种：

  1. 菊花链仲裁（Daisy Chain）：

    • 仲裁方式： 总线请求信号像链条一样依次传递给各个设备，优先级由设备在链上的位置决定，离仲裁器最近的设备优先级最高。
    • 特点：
      • 实现简单，硬件开销小。
      • 优先级固定，灵活性差。
      • 对电路故障敏感，任何一个设备的故障都可能影响整个仲裁过程。

  2. 计数器定时查询仲裁（Centralized, Polling）：

    • 仲裁方式： 仲裁器使用一个计数器来轮询各个设备，当某个设备发出总线请求时，仲裁器停止计数，并将总线分配给该设备。
    • 特点：
      • 优先级可变，可以通过改变计数器的起始值来调整优先级。
      • 硬件开销比菊花链略大。
      • 对电路故障的敏感性比菊花链低。

  3. 独立请求仲裁（Independent Request）：

    • 仲裁方式： 每个设备都有一条独立的请求线连接到仲裁器，仲裁器根据预定的优先级策略来决定将总线分配给哪个设备。
    • 特点：
      • 优先级灵活，可以根据需要设置不同的优先级策略。
      • 仲裁速度快。
      • 硬件开销最大，需要大量的控制线。

问题5：CPU 与外设的信息交换方式主要有哪几种？各具什么特点？

CPU 与外设的信息交换方式主要有以下几种：

  1. 程序查询方式（Programmed I/O）：

    • 方式： CPU 通过执行程序来周期性地查询外设的状态，判断外设是否准备好进行数据传输。
    • 特点：
      • 实现简单，不需要额外的硬件支持。
      • CPU 效率低，需要花费大量时间进行查询，浪费CPU资源。

  2. 中断方式（Interrupt）：

    • 方式： 外设在准备好进行数据传输时，向CPU发出中断请求，CPU 暂停当前的工作，转去处理中断，完成数据传输后，再返回到原来的工作。
    • 特点：
      • CPU 效率高，只有在外设需要服务时才占用CPU时间。
      • 需要中断控制器等硬件支持。

  3. 直接存储器访问方式（DMA, Direct Memory Access）：

    • 方式： 由专门的 DMA 控制器来管理外设和内存之间的数据传输，不需要 CPU 的干预。CPU 只需在传输开始前设置好 DMA 控制器，传输完成后由 DMA 控制器发出中断通知CPU。
    • 特点：
      • 传输速度快，效率高，大大减轻了CPU的负担。
      • 需要 DMA 控制器等硬件支持。
